Installation & Setup

System Requirements

  • Node.js 18+ (recommended: 20+)
  • PostgreSQL 16+
  • Redis 7+
  • Operating System: Linux, macOS, or Windows
  • Memory: Minimum 2GB RAM, 4GB+ recommended
  • Storage: 10GB+ for application and database

Installation Steps

1. Clone Repository

git clone <repository-url>
cd mero-jugx

2. Run Setup

npm run setup

The setup script will prompt you to choose:

  • Manual Setup: Requires local PostgreSQL and Redis installations
  • Docker Setup: Uses Docker Compose for PostgreSQL and Redis (recommended)

What setup does:

  • Installs all dependencies (backend and frontend)
  • Creates comprehensive .env files with all values (allows project to work immediately)
  • Starts Docker containers (if using Docker setup)
  • Runs all database migrations (creates all tables)
  • Seeds all initial data (packages, permissions, roles, etc.)

Alternative setup commands:

  • npm run setup:github - Direct Docker setup (bypasses prompt)
  • npm run setup:manual - Direct manual setup (bypasses prompt)

3. Start Application

# Development (interactive - choose dev or prod)
npm run dev

# Development (backend only, watch mode)
npm run start:dev

# Production (builds and starts both)
npm run start:prod

Complete Reset

To completely reset the system and remove everything:

npm run reset:all

What gets removed:

  • All node_modules (backend and frontend)
  • All dist and build folders
  • ALL database tables and data
  • All .env files
  • npm cache and logs
  • Docker containers (optional volume removal)

After reset: Run npm run setup to set up everything fresh.

For new installations: Clone the repository and run npm run setup - same process as after reset.

Database Management

Migrations

Run Migrations API

CLI Command:

npm run migration:run

What it does: Runs all pending database migrations.

Revert Migration

npm run migration:revert

What it does: Reverts the last executed migration.

Generate Migration

What it does: Generates new migration file based on entity changes.

Validate Migrations

Initialize Database

npm run db:init

What it does: Runs migrations and seeds initial data.

Reset Database

npm run db:reset
WARNING: This will drop all tables and data. Use with caution!

What it does: Drops database, recreates, runs migrations, and seeds data.

Check Database Connection

npm run db:check

What it does: Tests database connection and configuration.

Backup & Restore

Backup Database

# PostgreSQL backup
pg_dump -U postgres -d mero_jugx -f backup_$(date +%Y%m%d).sql

# With compression
pg_dump -U postgres -d mero_jugx | gzip > backup_$(date +%Y%m%d).sql.gz

Restore Database

# From SQL file
psql -U postgres -d mero_jugx -f backup_20241201.sql

# From compressed backup
gunzip -c backup_20241201.sql.gz | psql -U postgres -d mero_jugx

User Management

System User Operations

List All Users API

GET /users?page=1&limit=50
Authorization: Bearer <admin_token>

Required Permission: users.view

What it does: Returns paginated list of all users in organization.

Get User Details API

GET /users/:id
Authorization: Bearer <admin_token>

What it does: Returns complete user details including all organizations.

Update User (Admin) API

PUT /users/:id
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "first_name": "John",
  "last_name": "Doe",
  "email": "newemail@example.com",
  "status": "active|suspended|inactive"
}

Required Permission: users.edit

What it does: Admin can update any user's information including status.

Revoke User Access API

POST /users/:id/revoke
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "reason": "Terms violation",
  "transfer_data": true,
  "transfer_to_user_id": "uuid"
}

Required Permission: users.revoke

What it does: Revokes user access across all organizations, optionally transfers data.

Impersonate User API

POST /users/:id/impersonate
Authorization: Bearer <admin_token>

Required Permission: Owner/Admin or users.impersonate

What it does: Allows admin to impersonate user for support purposes.

Bulk Operations

Export Users API

GET /data-management/users/export
Authorization: Bearer <admin_token>

What it does: Exports all users to CSV for backup or analysis.

Import Users API

POST /data-management/users/import

{
  "csv_data": "email,first_name,last_name,role\nuser@example.com,John,Doe,member",

Bulk Assign Role API

POST /data-management/users/bulk-assign-role

Bulk Send Invitations API

POST /data-management/invitations/bulk-send
Authorization: Bearer <admin_token>

{

What it does: Sends invitations to multiple emails.

Backup & Recovery

Backup Strategy

  1. Database Backups: Daily automated backups
  2. File Backups: Backup uploaded files directory
  3. Configuration Backups: Backup .env and config files
  4. Offsite Storage: Store backups in separate location

Automated Backup Script

#!/bin/bash
# backup.sh
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="./backups"

# Database backup
pg_dump -U postgres mero_jugx | gzip > "$BACKUP_DIR/db_$DATE.sql.gz"

# Files backup
tar -czf "$BACKUP_DIR/files_$DATE.tar.gz" ./uploads

# Config backup
cp .env "$BACKUP_DIR/env_$DATE.backup"

# Clean old backups (keep last 30 days)
find "$BACKUP_DIR" -type f -mtime +30 -delete

Recovery Procedures

Database Recovery

  1. Stop application
  2. Restore database from backup
  3. Run any pending migrations
  4. Verify data integrity
  5. Restart application

File Recovery

  1. Extract backup archive
  2. Restore files to uploads directory
  3. Verify file permissions

Troubleshooting

Common Issues

Application Won't Start

  • Check Node.js version: node -v
  • Verify database connection
  • Check Redis is running
  • Review error logs
  • Check port availability

Database Connection Errors

  • Verify PostgreSQL is running
  • Check database credentials in .env
  • Test connection: psql -U postgres -d mero_jugx
  • Check firewall settings
  • Verify database exists

Redis Connection Errors

  • Check Redis is running: redis-cli ping
  • Verify Redis credentials
  • Check Redis port (default: 6379)
  • Review Redis logs

High Memory Usage

  • Check for memory leaks
  • Review active connections
  • Optimize database queries
  • Clear Redis cache if needed
  • Restart application

Slow Performance

  • Check database query performance
  • Review index usage
  • Check Redis cache hit rate
  • Review application logs
  • Monitor system resources

Log Files

Log files are typically located in:

  • Application logs: logs/app.log
  • Error logs: logs/error.log
  • Access logs: logs/access.log

Cleanup Tasks

Clean Expired Sessions

Expired sessions are automatically cleaned, but you can manually trigger:

# Clean expired Redis keys
redis-cli --scan --pattern "session:*" | xargs redis-cli DEL

Archive Old Audit Logs

Archive audit logs older than retention period (configure in settings).

Clean Temporary Files

# Clean upload temp directory
find ./uploads/temp -type f -mtime +7 -delete
