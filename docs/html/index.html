Main Interactive Scripts

  • npm run setup - Interactive setup
    • Prompts: Manual Setup or Docker Setup
    • Installs dependencies, creates .env files, initializes database
    • Creates all tables and seeds all data
  • npm run reset - Interactive reset
    • Prompts: Everything, Database Only, Frontend Build Only, or Backend Build Only
    • Database reset: Drops all tables, recreates them, and seeds data
  • npm run run - Interactive run
    • Prompts: Development Server or Production Server
    • Development: Starts backend and frontend with hot reload
    • Production: Builds and starts both for production
  • npm run test - Run all tests
  • npm run ci - CI/CD workflow (prompts for branch selection)

Database Commands

  • npm run db:init - Initialize database
    • Creates all tables (runs migrations)
    • Seeds all initial data (packages, permissions, roles, package features, role templates)
    • Safe to run multiple times (skips if already done)
  • npm run db:reset - Reset database (‚ö†Ô∏è DANGEROUS - deletes all data)
    • Drops all existing tables and types
    • Clears migrations table
    • Recreates all tables fresh (runs migrations)
    • Seeds all initial data
  • npm run db:check - Check database connection and status
  • npm run migration:generate -- -n MigrationName - Generate new migration
  • npm run migration:run - Run pending migrations
  • npm run migration:revert - Revert last migration
  • npm run migration:show - Show migration status
  • npm run seed:run - Run database seeds manually

Build & Start Commands

  • npm run build - Build backend only
  • npm run build:all - Build both backend and frontend
  • npm run start:dev - Start backend in watch mode
  • npm run start:prod - Start production server (requires build first)

Docker Commands

  • npm run docker:up - Start development containers (PostgreSQL, Redis)
  • npm run docker:down - Stop development containers
  • npm run docker:logs - View container logs (follow mode)
  • npm run docker:build - Build development Docker images

Branch Management Commands

  • npm run branch:create - Create feature branch
    • Prompts for parent branch (development/testing/production)
    • Prompts for branch type (feature/bugfix/hotfix)
    • Creates properly named branch (e.g., development/feature-user-auth)
  • npm run branch:check - Check branch permissions
    • Shows which branches you have access to
    • Displays current branch status
  • npm run branch:list - List feature branches
    • Lists all feature branches for your assigned parent branch

Branching Strategy

This project uses a protected branching strategy:

  • main - üîí Locked (Repository owner only)
  • development - üîí Protected (Requires feature branches)
  • testing - üîí Protected (Requires feature branches)
  • production - üîí Protected (Requires feature branches)

Workflow:

  1. Get branch access from repository owner
  2. Create feature branch: npm run branch:create
  3. Make changes and commit
  4. Push to feature branch (CI/CD runs automatically)
  5. Create PR to parent branch (development/testing/production)
  6. After approval, merge to parent branch
  7. Owner merges to main when ready

Environment Variables

All setup scripts automatically create comprehensive .env files with defaults that allow the project to work immediately.

  • Backend .env: Application, Database, Redis, JWT, Email, Payment gateways, and more
  • Frontend frontend/.env: API URL, App name, Sentry, Currency settings
  • Default Values: All variables have defaults - project works out of the box
  • eSewa Test Credentials: Included by default (works immediately)
  • JWT Secrets: Auto-generated during setup

What Gets Seeded

When you run db:init or db:reset, the following data is automatically seeded:

  • Packages - Subscription packages with features and limits
  • Permissions - System permissions for RBAC
  • Roles - Default roles (admin, user, etc.) with permissions
  • Package Features - Features associated with each package
  • Role Templates - Pre-configured role templates
