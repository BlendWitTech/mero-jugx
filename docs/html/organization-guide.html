Setup & Reset

This section covers how to set up and reset the Mero Jugx project, which is essential for organization administrators and developers.

Complete Reset

To completely reset the project and remove everything:

npm run reset:all

What gets removed:

  • All node_modules (backend and frontend)
  • All dist and build folders
  • ALL database tables and data
  • All .env files
  • npm cache and logs
  • Docker containers (optional volume removal)

After reset: Run npm run setup to set up everything fresh.

Setup

To set up the project (for new developers or after reset):

npm run setup

The setup script will prompt you to choose:

  • Manual Setup: Requires local PostgreSQL and Redis installations
  • Docker Setup: Uses Docker Compose for PostgreSQL and Redis (recommended)

What setup does:

  • Installs all dependencies (backend and frontend)
  • Creates comprehensive .env files with all values (allows project to work immediately)
  • Starts Docker containers (if using Docker setup)
  • Runs all database migrations (creates all tables)
  • Seeds all initial data (packages, permissions, roles, etc.)

For new developers: Clone the repository and run npm run setup - same process as after reset.

Organization Overview

Organizations are workspaces where teams collaborate. Each organization has:

  • Members with assigned roles
  • A subscription package with features and limits
  • Settings and branding customization
  • Documents and resources
  • Billing and payment history

Member Management

Adding Members

Invite users to join your organization via email invitations.

Create Invitation API

POST /invitations
Authorization: Bearer <token>
Content-Type: application/json

{
  "email": "newuser@example.com",
  "role_id": 2
}

Response: 201 Created

{
  "id": "invitation_uuid",
  "email": "newuser@example.com",
  "role_id": 2,
  "token": "invitation_token",
  "status": "pending",
  "expires_at": "2024-12-31T23:59:59Z"
}

What it does: Creates invitation, generates secure token, sends email with invitation link.

Bulk Send Invitations API

POST /data-management/invitations/bulk-send
Authorization: Bearer <token>
Content-Type: application/json

{
  "emails": ["user1@example.com", "user2@example.com"],
  "role_id": 2
}

Required Permission: invitations.create

What it does: Sends invitations to multiple emails at once.

List Invitations API

GET /invitations?page=1&limit=10&status=pending
Authorization: Bearer <token>

Query Parameters:

  • page - Page number
  • limit - Items per page
  • status - Filter by status (pending, accepted, expired)

Cancel Invitation API

DELETE /invitations/:id
Authorization: Bearer <token>

What it does: Cancels pending invitation, prevents acceptance.

Managing Members

List Organization Users API

GET /users?page=1&limit=10&search=john
Authorization: Bearer <token>

Required Permission: users.view

What it does: Returns all members in organization with pagination and search.

Get User Details API

GET /users/:id
Authorization: Bearer <token>

What it does: Returns user details including role and membership status.

Update User (Admin) API

PUT /users/:id
Authorization: Bearer <token>
Content-Type: application/json

{
  "first_name": "John",
  "last_name": "Doe",
  "email": "newemail@example.com",
  "status": "active"
}

Required Permission: users.edit

What it does: Updates user information (admin only).

Revoke User Access API

POST /users/:id/revoke
Authorization: Bearer <token>
Content-Type: application/json

{
  "reason": "Violation of terms",
  "transfer_data": true,
  "transfer_to_user_id": "uuid"
}

Required Permission: users.revoke

What it does: Removes user from organization, optionally transfers their data to another user.

Bulk Assign Role API

POST /data-management/users/bulk-assign-role
Authorization: Bearer <token>
Content-Type: application/json

{
  "user_ids": ["uuid1", "uuid2", "uuid3"],
  "role_id": 2
}

Required Permission: users.edit

What it does: Assigns role to multiple users at once.

Export Users API

GET /data-management/users/export
Authorization: Bearer <token>

What it does: Exports all organization users to CSV file.

Import Users API

POST /data-management/users/import
Authorization: Bearer <token>
Content-Type: application/json

{
  "csv_data": "email,first_name,last_name\nuser@example.com,John,Doe",
  "role_id": 2
}

What it does: Imports users from CSV, creates accounts or sends invitations.

Roles & Permissions

Managing Roles

List Roles API

GET /roles?page=1&limit=10
Authorization: Bearer <token>

Required Permission: roles.view

What it does: Returns all roles in organization.

Get Role by ID API

GET /roles/:id
Authorization: Bearer <token>

Response includes: Role name, description, permissions, user count.

Create Role API

POST /roles
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Project Manager",
  "slug": "project-manager",
  "description": "Manages projects and team",
  "permissions": [
    "projects.view",
    "projects.edit",
    "tasks.view",
    "tasks.edit",
    "users.view"
  ]
}

Required Permission: roles.create

What it does: Creates custom role with specified permissions.

Update Role API

PUT /roles/:id
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Senior Project Manager",
  "description": "Updated description",
  "permissions": ["projects.view", "projects.edit", "projects.delete"]
}

Required Permission: roles.edit

Note: System roles (Owner, Admin, Member) cannot be modified.

Delete Role API

DELETE /roles/:id
Authorization: Bearer <token>

Required Permission: roles.delete

What it does: Deletes role if not assigned to users and not a system role.

Get All Permissions API

GET /roles/permissions
Authorization: Bearer <token>

What it does: Returns all available permissions grouped by resource.

Response format:

{
  "users": ["view", "edit", "delete", "revoke"],
  "organizations": ["view", "edit", "settings"],
  "roles": ["view", "create", "edit", "delete"],
  "packages": ["view", "upgrade"],
  "chat": ["view", "send", "manage_group"],
  ...
}

Get Role Usage Counts API

GET /roles/usage-counts
Authorization: Bearer <token>

What it does: Returns count of users assigned to each role.

Assign Role to User API

PUT /user-roles/:userId/role
Authorization: Bearer <token>
Content-Type: application/json

{
  "role_id": 2
}

What it does: Assigns or changes user's role in organization.

Role Templates

Create Role from Template API

Packages & Subscriptions

Viewing Packages

List Available Packages API

GET /packages
Authorization: Bearer <token>

Response includes: All available packages with pricing, features, and limits.

Get Package by ID API

GET /packages/:id
Authorization: Bearer <token>

What it does: Returns detailed package information.

List Package Features API

GET /packages/features
Authorization: Bearer <token>

What it does: Returns all available features that can be purchased separately.

Current Package

Get Current Package API

GET /organizations/me/package
Authorization: Bearer <token>

Response includes: Current package, limits, usage, expiration date, auto-renewal status.

Upgrade/Downgrade Package API

PUT /organizations/me/package
Authorization: Bearer <token>
Content-Type: application/json

{
  "package_id": 3,
  "period": "1_year"
}

Required Permission: packages.upgrade

What it does: Changes organization package. Validates usage doesn't exceed new limits.

Period options: 1_month, 3_months, 6_months, 1_year

Calculate Upgrade Price API

POST /organizations/me/package/calculate-upgrade-price
Authorization: Bearer <token>
Content-Type: application/json

{
  "package_id": 3,
  "period": "1_year",
  "custom_months": 12
}

What it does: Calculates prorated upgrade price with credit for remaining time on current package.

POST /organizations/me/features
Authorization: Bearer <token>

Best Practices

Member Management

  • Always assign appropriate roles when inviting members
  • Use bulk operations for managing multiple members
  • Regularly review member access and permissions
  • Export member lists for backups

Role Management

  • Create roles based on job functions, not individuals
  • Follow principle of least privilege
  • Review role usage before deleting
  • Use role templates for common roles

Package Management

  • Monitor usage against package limits
  • Calculate upgrade costs before upgrading
  • Set up auto-renewal for convenience
  • Keep payment methods updated

Security

  • Enable MFA for organization
  • Regularly review audit logs
  • Revoke access for inactive members
  • Use secure payment methods
